# ReimburseFlow - 项目总结文档

> 最后更新：2026-04-23

---

## 一、项目概述

**ReimburseFlow** 是一个个人报销准备工具，帮助用户快速记录消费、管理发票、准备报销材料。

| 项目 | 说明 |
|------|------|
| 定位 | 个人报销辅助工具（MVP 阶段） |
| 技术方案 | Web App（纯前端，无后端） |
| 访问地址 | https://terazhou.github.io/ReimburseFlow/ |
| 代码仓库 | https://github.com/TeraZhou/ReimburseFlow |
| 费用 | 完全免费 |
| 数据存储 | 浏览器本地（IndexedDB）+ GitHub Gist 云备份 |
| 离线使用 | 支持（首次 OCR 需联网下载引擎） |

## 二、功能清单

### 1. 首页概览

- [x] 本月概览：交易和发票的总额 / 已报销 / 未报销金额
- [x] 历史概览：全部交易和发票的总额 / 已报销 / 未报销金额
- [x] 快捷操作入口（2x2 网格）：手动记账、拍照记账、上传发票、导出记录
- [x] 最近 5 条交易记录，可点击跳转详情

### 2. 交易记录

#### 列表页
- [x] 双重下拉筛选：费用类别 + 公司抬头，可叠加筛选
- [x] 日期范围筛选：起始日期 + 截止日期，快速定位特定时间段
- [x] 平铺列表显示（按日期降序排列）
- [x] 底部汇总栏：合计金额、已报销、未报销（随筛选实时更新）

#### 批量操作
- [x] 批量管理模式：点击 header「管理」按钮进入，每条记录前显示复选框
- [x] 全选 / 取消全选
- [x] 批量标记已报销
- [x] 批量标记未报销
- [x] 批量删除（带二次确认弹窗）
- [x] 已选数量实时显示

#### 录入与编辑
- [x] 手动录入交易（日期、金额、分类、公司抬头、备注、最多 3 张凭证图片）
- [x] 拍照/选图录入交易（OCR 自动识别金额、日期、分类）
- [x] 公司抬头从预设列表下拉选择（在设置页维护）
- [x] 交易详情查看（含凭证原图放大）
- [x] 交易编辑、删除
- [x] 一键切换已报销/未报销状态（列表页 Badge 或详情页均可操作）

### 3. 发票管理

#### 列表页
- [x] 三重下拉筛选：状态 + 费用类别 + 公司抬头，可叠加筛选
- [x] 日期范围筛选：起始日期 + 截止日期
- [x] 底部汇总栏：合计金额、已报销、未报销、发票张数
- [x] 发票卡片展示：缩略图（图片/PDF）、金额、公司抬头、发票号码、日期、状态
- [x] 批量下载附件（当前筛选结果内的所有附件，500ms 间隔逐个下载）
- [x] 一键切换已报销/未报销状态

#### 批量操作
- [x] 批量管理模式：点击 header「管理」按钮进入
- [x] 全选 / 取消全选
- [x] 批量标记已报销
- [x] 批量标记未报销
- [x] 已选数量实时显示
- [x] 不提供批量删除（防止误操作）

#### 录入与编辑
- [x] 发票新增（拍照/选图 + OCR 自动识别）
- [x] 支持 JPG、PNG、PDF 格式附件（PDF 不压缩、不 OCR）
- [x] 公司抬头从预设列表下拉选择（在设置页维护，必填）
- [x] 发票详情查看（含原图放大、PDF 下载）
- [x] 发票编辑、删除
- [x] 发票号码去重提示

### 4. 分类管理

- [x] 8 个内置默认分类（不可删除）：交通费、餐饮费、住宿费、通讯费、办公用品、差旅费、招待费、其他
- [x] 用户自定义分类（新增、编辑、删除）
- [x] 分类名称唯一性校验

### 5. 公司抬头管理

- [x] 预设公司抬头列表（新增、编辑、删除）
- [x] 名称唯一性校验
- [x] 交易/发票填写页通过下拉选择公司抬头
- [x] 云备份包含公司抬头数据

### 6. OCR 识别

- [x] 基于 Tesseract.js v5 的端侧 OCR（中文 + 英文）
- [x] 图片预处理（灰度 + 对比度增强，最大 2000px）
- [x] 识别字段：
  - 金额（7 级优先级：价税合计 > 合计/总计 > 应收/应付 > 货币符号 > 金额关键词 > 英文关键词 > 独立数字）
  - 日期（支持 YYYY年MM月DD日、YYYY-MM-DD、YYYY/MM/DD、YY-MM-DD、YYYYMMDD 等）
  - 发票号码（关键词 + 最长数字串回退）
  - 销售方名称（关键词 + 第二个公司名回退）
  - 分类推断（关键词映射，如"餐"→餐饮费、"滴滴"→交通费）
- [x] 公司抬头不再由 OCR 识别，改为从预设列表选择
- [x] 识别结果预填表单，用户可对照原图修改
- [x] 查看原始 OCR 识别文本（调试用）
- [x] OCR 引擎按需加载（首次使用时从 CDN 加载）

### 7. 云备份与恢复

- [x] 基于 GitHub Gist API 的云备份（用户提供 GitHub Personal Access Token）
- [x] 手动备份到云端（创建/更新私有 Gist）
- [x] 从云端恢复数据（覆盖当前所有数据，带确认弹窗）
- [x] 自动备份：每次数据变更后自动触发（5 秒防抖，静默失败）
- [x] 跨设备 Gist 查找：本地 Gist ID 丢失时自动搜索已有备份
- [x] 备份时间戳显示
- [x] 备份数据包含：分类、交易、发票、公司抬头
- [x] 注意：备份数据不含图片，仅含文本/数值字段

### 8. 数据导出

- [x] 交易记录导出为 CSV（日期、分类、金额、公司抬头、备注、报销状态、是否有凭证）
- [x] 发票记录导出为 CSV（发票号码、金额、开票日期、公司抬头、销售方、报销状态、备注）
- [x] UTF-8 BOM 前缀，Excel 直接打开无乱码
- [x] 首页快捷导出入口（弹窗选择交易/发票）
- [x] 交易列表页导出按钮（导出全部交易）
- [x] 发票列表页导出按钮（导出当前筛选结果）

### 9. PWA 支持

- [x] 添加到手机主屏幕，全屏运行无浏览器地址栏
- [x] 持久化存储请求（防止浏览器自动清理 IndexedDB）
- [x] 未获持久化权限时一次性提示用户添加到主屏幕

## 三、OCR 识别能力说明

### 推荐使用方式
**从相册选择清晰照片** >> 拍照。相册照片通常光线充足、对焦清晰，识别率显著高于实时拍照。

### 各字段识别情况（相册选图）

| 字段 | 识别可靠性 | 说明 |
|------|-----------|------|
| 发票号码 | 高 | 20 位数字串，通常能准确识别 |
| 日期 | 中高 | 支持"年月日"格式及纯数字格式 |
| 分类 | 中 | 基于关键词匹配（"餐"→餐饮费等） |
| 金额 | 中低 | 取决于发票上金额区域的识别质量 |
| 销售方名称 | 中 | 类似公司抬头 |

### 拍照识别
拍照受光线、角度、手抖影响，识别准确率明显低于相册选图。建议先拍照保存到相册，再从相册选择。

### 局限性
- Tesseract.js 对中文发票的识别质量有上限
- 复杂排版、模糊、反光的图片识别效果差
- 部分字段可能识别不到，需手动补齐

### 数据流概览

1. **所有数据**存储在 IndexedDB（4 个 store：category、transaction、invoice、company_title）
2. **图片**以 Base64 Data URI 存入数据库（上传时压缩至 1200px / JPEG 0.7 质量）
3. **PDF 附件**原样存储，不压缩、不 OCR
4. **每次数据变更**后自动触发 GitHub Gist 备份（5 秒防抖，不含图片）
5. **CSV 导出**支持交易和发票两种格式，UTF-8 BOM 确保 Excel 兼容

### 依赖说明

| 依赖 | 用途 | 加载方式 |
|------|------|---------|
| Tesseract.js v5 | OCR 文字识别 | CDN 动态加载（首次使用时） |

无其他外部依赖，纯原生 HTML/CSS/JavaScript。

## 五、部署方式

- **平台**：GitHub Pages（免费）
- **方式**：代码放在 `docs/` 目录，GitHub Pages 设置 Source 为 `main` 分支 `/docs` 目录
- **更新流程**：本地修改 → `git push` → 1-2 分钟自动部署生效
- **缓存管理**：通过文件引用后的 `?v=N` 参数控制版本（当前 v12）

## 六、已知限制

1. **云备份不含图片**：凭证图片和发票附件体积较大，仅备份文本/数值字段
2. **OCR 质量上限**：Tesseract.js 中文发票识别准确率有限，部分字段需手动补齐
3. **图片存储**：凭证图片以 Base64 存在 IndexedDB 中，大量图片可能导致存储压力
4. **跨设备数据**：云备份不含图片，换设备恢复后凭证/发票图片会丢失
