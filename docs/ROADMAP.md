## 第二期：数据安全 + 体验优化

**目标**：数据不丢失，用起来更顺手

### 2.1 数据持久化（优先级最高）
- [x] 导出/导入完整数据（JSON 格式，通过 GitHub Gist 云备份）
  - 支持导出所有交易、发票、分类、公司抬头
  - 支持从云端恢复数据
  - 自动备份（5 秒防抖）
- [ ] 定期自动提醒备份
  - 每周/每月弹窗提醒"是否备份数据"

### 2.2 体验优化
- [x] 交易/发票列表筛选优化（费用类别 + 公司抬头双重下拉筛选）
- [x] 发票查重（输入发票号码时提示重复）
- [x] 发票批量管理模式（全选/标记已报销/未报销）
- [x] 公司抬头预设管理（从自由文本改为预设列表 + 下拉选择）
- [ ] 交易记录搜索功能（按备注、金额搜索）
- [ ] 长按交易列表项快速删除
- [ ] 金额输入支持计算器（如输入"35+12+8"自动计算）

### 2.3 图片优化
- [ ] 图片存储改用 IndexedDB 单独存储（与文本数据分离）
- [ ] 图片压缩质量控制（大图自动缩到合理尺寸）
- [ ] 图片懒加载（列表页不加载原图）

**状态**：大部分完成，剩余搜索和图片优化待做

## 第三期：云端同步 + 多设备

**目标**：换设备不丢数据，多端同步

### 3.1 云同步方案
可选方向（按推荐顺序）：

| 方案 | 优点 | 缺点 |
|------|------|------|
| Firebase（Google） | 免费额度大，实时同步 | 需 Google 账号，国内可能需梯子 |
| Supabase | 开源、免费额度够用 | 需注册 |
| 简易后端（Cloudflare Workers + KV） | 完全免费，国内可访问 | 需要开发 |

- [ ] 用户登录（简单的 PIN 码或账号密码）
- [ ] 数据自动同步到云端
- [ ] 多设备查看同一份数据

### 3.2 数据分享
- [ ] 导出报销汇总 PDF（含交易明细 + 发票图片）
- [ ] 一键生成报销单（按公司模板格式）

**预计工作量**：7-10 天

---

## 第四期：智能化 + 高级功能

**目标**：减少重复操作，提升报销效率

### 4.1 OCR 升级
- [ ] 接入更好的 OCR 引擎（当有免费方案可用时）
- [ ] 发票模板识别（区分增值税普票、专票、电子发票）
- [ ] 识别后自动校验（金额 = 单价 × 数量 × (1+税率)）

### 4.2 报销流程
- [ ] 项目/出差标签（给交易打标签，按项目汇总）
- [ ] 报销规则校验（如"餐饮费单次不超过 200 元"）
- [ ] 报销单生成（选择交易 → 生成报销汇总）

### 4.3 数据分析
- [ ] 月度/年度消费统计图表
- [ ] 分类占比分析
- [ ] 报销效率统计（从消费到报销的天数）

**预计工作量**：10-15 天
